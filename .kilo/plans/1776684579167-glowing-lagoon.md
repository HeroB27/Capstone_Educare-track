# Wizard Step Reorganization: Add Parent & Student Flow

## 1. Current Flow Analysis

### Existing Steps (6 total)
- **Step 1**: Parent Information (name, address, role, phone)
- **Step 2**: Account Creation (username, password)
- **Step 3**: Parent Confirmation & Save (saves parent to DB)
- **Step 4**: Student Information (dynamic forms with photo upload)
- **Step 5**: Student Confirmation (summary of each student with photo preview)
- **Step 6**: ID Card Preview (final review before completion)

### Data Save Points
- Parent saved when moving from Step 3 → Step 4 (`saveParentToDB()`)
- Students saved when moving from Step 5 → Step 6 (`saveStudentsToDB()`) — includes photo upload to Supabase Storage
- Final completion on Step 6 closes modal and refreshes table

### Photo Handling
- Photos are captured as Data URLs (via file input or webcam) and stored in `studentsPhotos[formId]`
- During `saveStudentsToDB()`, photos are uploaded to Supabase Storage and `profile_photo_url` is saved to the students table
- ID Preview uses Data URL from `studentsPhotos` for immediate display, not DB URL

---

## 2. Requested Changes (Finalized)

**Chosen approach: Option B – Batch Grid Preview**

1. **Add Confirmation Modal after Step 4**
   - When clicking "Next Step" on the Student Information step, show a modal: *"Do you confirm the details of the parent and student(s)?"*
   - Buttons: **"Back to Edit"** (close modal, stay on step 4) and **"Yes, Continue"** (proceed to save and preview)

2. **Remove Step 5 (Student Confirmation) entirely**
   - The Student Summary step with dropdown selector is eliminated.

3. **Retain Step 6 as Step 5: ID Preview (now GRID view)**
   - After confirmation and save, show a **grid of ID cards** for ALL students at once (no selector).
   - Each ID card displays the student's photo, name, ID number, QR, etc.
   - Layout: Responsive grid (1 column if 1 student, 2×N grid if 2–4 students, etc.)
   - Provide **"Print All IDs"** and **"Complete Enrollment"** buttons.

4. **Adjust Step Count**
   - Total steps: **6 → 5**
   - New steps: 1=Parent Info, 2=Account, 3=Parent Confirm, 4=Student Info, 5=ID Preview Grid + Complete.

---

## **Decision: Option B – Batch Grid Preview** ✅

User selected **Option B** (grid of all IDs shown at once after confirmation + save). This enables batch review and batch printing.

  ```javascript
  function printAllIDCards() {
      const grid = document.getElementById('id-cards-grid');
      if (!grid?.innerHTML?.trim()) {
          showNotification('No ID cards to print', 'error');
          return;
      }
      const cards = grid.querySelectorAll('[class*="w-\\[2in\\]"]');
      if (cards.length === 0) {
          showNotification('No ID cards to print', 'error');
          return;
      }
      const printWindow = window.open('', '_blank');
      if (!printWindow) {
          showNotification('Please allow pop-ups to print', 'error');
          return;

      const cardsHtml = Array.from(cards).map(card => card.outerHTML).join('<div class="id-page-break"></div>');
      printWindow.document.write(`
          <!DOCTYPE html>
          <html>
          <head>
              <meta charset="UTF-8">
              <title>Student ID Cards</title>
              <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;700;900&display=swap" rel="stylesheet">
              <script src="https://cdn.tailwindcss.com"></script>

                  [class*="w-\\[2in\\]"] { width: 2in !important; }
                  [class*="h-\\[3in\\]"] { height: 3in !important; }
                  * { -webkit-print-color-adjust: exact !important; print-color-adjust: exact !important; }
                  body { font-family: 'Inter', sans-serif; margin: 0; padding: 20px; display: flex; flex-direction: column; align-items: center; gap: 0.5in; }
  ```

## 5. Edge Cases & Recommendations

### 5.1 Navigation After Save
- Once students are saved (`studentsSavedToDB = true`):
  - If user clicks **Previous** from Step 5 to Step 4, then clicks **Next** again, the confirmation modal will **not** appear (flag is true), and the wizard proceeds directly to Step 5 **without saving changes**.
  - This mirrors the original behavior (after Step 5 save, going back did not trigger another save).
  - Recommendation: To prevent confusion, either:
    - Disable the **Previous** button on Step 5 (ID Preview), OR
    - Show a warning that changes made after saving will not be persisted, OR
    - Reset `studentsSavedToDB` when returning to Step 4 and allow updates (requires handling updates vs inserts – out of scope for this change).

  Given the original design, keeping the same behavior is acceptable.

### 5.2 Loading State
- `saveStudentsToDB()` involves network I/O (DB insert + Storage uploads). The UI may appear frozen.
  - **Recommendation**: Add a spinner/loading overlay on the modal during save. (Future enhancement.)

### 5.3 Error Handling
- If photo upload fails, the student record is still created without a photo. The ID preview will show placeholder.
  - Already handled with try/catch; user sees a notification.
  - Consider making photo required if essential (validation). (Optional.)

### 5.4 Confirmation Content
- The confirmation modal currently has generic text. Could also summarize parent name and number of students.
  - Possible enhancement: include mini-summary in modal.

### 10.2 Debug Utility Functions (add to JS)

```javascript
// ==================== DEBUG UTILITIES ====================
let debugPanelVisible = false;
const debugLogs = [];

function logDebug(message, data = null) {
    const timestamp = new Date().toISOString().split('T')[1].slice(0, -1);
    let entry = `[${timestamp}] ${message}`;
    if (data !== null) {
        entry += ` | ${typeof data === 'object' ? JSON.stringify(data, null, 2) : data}`;
    }
    debugLogs.push(entry);
    if (debugPanelVisible) {
        const output = document.getElementById('debug-output');
        if (output) output.textContent = debugLogs.join('\n');
    }
}

function clearDebugLog() {
    debugLogs.length = 0;
    const output = document.getElementById('debug-output');
    if (output) output.textContent = '';
}

function toggleDebugPanel() {
    const panel = document.getElementById('debug-panel');
    if (!panel) return;
    debugPanelVisible = !debugPanelVisible;
    panel.classList.toggle('hidden', !debugPanelVisible);
}

function shouldShowDebug() {
    return window.location.search.includes('debug=1') || window.location.hostname === 'localhost';
}

// Initialize on load
function initDebugConsole() {
    if (shouldShowDebug()) {
        debugPanelVisible = true;
        const panel = document.getElementById('debug-panel');
        if (panel) panel.classList.remove('hidden');
        logDebug('Debug console initialized');
        logDebug('Environment:', window.location.hostname);
    }

    // Keyboard shortcut: Ctrl+Shift+D
    document.addEventListener('keydown', (e) => {
        if (e.ctrlKey && e.shiftKey && e.key === 'D') {
            e.preventDefault();
            toggleDebugPanel();
        }
    });
}
```
