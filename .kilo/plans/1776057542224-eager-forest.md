# Announcements Photo Upload - Implementation Plan

## Overview
Allow admin and teachers to upload photos/posters with their announcements, viewable by all modules (guard, clinic, parent).

---

## Step 1: Database Schema Update

Add `image_url` column to the `announcements` table:

```sql
ALTER TABLE public.announcements ADD COLUMN image_url text;
```

---

## Step 2: Supabase Storage Bucket Creation (No RLS)

Run this SQL to create the bucket:

```sql
-- Create storage bucket for announcement photos
INSERT INTO storage.buckets (id, name, public, file_size_limit, allowed_mime_types)
VALUES (
  'announcement-photos',
  'announcement-photos',
  true,
  5242880,
  ARRAY['image/jpeg', 'image/png', 'image/gif', 'image/webp']
);
```

**Note:** Bucket is created with `public = true` - no RLS policies needed.

---

## Step 3: Admin Module Updates

**File: `admin/admin-announcements.html`**
- Add file input for photo upload in the modal form

**File: `admin/admin-announcements.js`**
- Update `saveAnnouncement()` to:
  1. Upload photo to `announcement-photos` bucket if selected
  2. Get public URL and include in payload as `image_url`
  3. Update existing display logic to show image if present

---

## Step 4: Teacher Module Updates

**File: `teacher/teacher-announcements-board.html`**
- Add file input for photo upload in the announcement form

**File: `teacher/teacher-core.js`**
- Update `postAnnouncement()` to:
  1. Upload photo to `announcement-photos` bucket if selected
  2. Get public URL and include in insert payload

---

## Step 5: Display Updates (All Modules)

Update announcement display logic in:
- `guard/guard-announcements-board.js`
- `clinic/clinic-announcements-board.js`
- `parent/parent-announcements-board.js`

Display the image below the content when `image_url` exists:
```html
${ann.image_url ? `<img src="${ann.image_url}" class="mt-3 rounded-lg max-w-full">` : ''}
```
