# Guard Pass System Implementation Plan

## Overview
Digital guard pass system for students leaving school premises with teacher permission, parent notifications, and admin oversight.

---

## Database Schema

### New Table: `guard_passes`

```sql
CREATE TABLE public.guard_passes (
  id bigint GENERATED ALWAYS AS IDENTITY NOT NULL,
  student_id bigint NOT NULL,
  teacher_id bigint NOT NULL,
  purpose text NOT NULL,
  time_out time without time zone NOT NULL,
  issued_at timestamp with time zone DEFAULT now(),
  status text DEFAULT 'Active'::text, -- Active, Used, Expired, Cancelled
  parent_notified boolean DEFAULT false,
  parent_notified_at timestamp with time zone,
  notes text,
  CONSTRAINT guard_passes_pkey PRIMARY KEY (id),
  CONSTRAINT guard_passes_student_id_fkey FOREIGN KEY (student_id) REFERENCES public.students(id),
  CONSTRAINT guard_passes_teacher_id_fkey FOREIGN KEY (teacher_id) REFERENCES public.teachers(id)
);
```

**No RLS (Row Level Security)** - as per user requirement.

## Teacher Module Updates

### File: `teacher/teacher-guard-pass.html`

**UI Components:**
- "Create Guard Pass" button in sidebar
- Guard Pass creation form:
  - Student dropdown (homeroom students only)
  - Purpose text field
  - Time out picker (default current time)
- Active passes list with status badges
- "Cancel" button for unused passes

### File: `teacher/teacher-guard-pass.js`

**Functions:**
- `loadGuardPasses()` - Fetch all passes for teacher's homeroom students
- `createGuardPass(studentId, purpose, timeOut)` - Create new pass
- `cancelGuardPass(passId)` - Cancel unused pass
- `notifyParent(passId)` - Send notification to parent
- `loadHomeroomStudents()` - Get enrolled students for dropdown

**Logic:**
1. Teacher selects student from homeroom dropdown
2. Enters purpose and time
3. On save: creates guard_pass record + sends parent notification
4. Parent receives notification via `notifications` table

## Parent Module Updates

### Notification Format
When guard pass is issued, parent receives:
```

Message: "A guard pass has been issued for [Student Name] to leave school premises.
Purpose: [purpose]
Time Out: [time]
Issued by: [Teacher Name]"
```

## Status Values

| Status | Description |
|--------|-------------|
| Active | Pass issued, not yet used |
| Used | Student has left premises |
| Expired | Not used within school hours |
| Cancelled | Teacher cancelled the pass |

---

## Notification Flow

```
Teacher creates pass
       ↓
guard_passes record created (status: Active)
       ↓
Parent notified (notifications table)
       ↓
Admin notified (admin_alerts table)
       ↓
Guard verifies at gate → marks as Used
```
