# Attendance Backend Sync Analysis: Current vs Expected

**Date:** 2026-04-20  
**Scope:** Compare current system implementation against the provided attendance logic and database schema

---

## Executive Summary

The current Educare attendance system implements **~60-70%** of the expected logic. Core infrastructure (gate scanning, basic homeroom/subject attendance, notifications) is functional, but several critical gaps exist in:

1. **attendance_daily_summary** population is incomplete
2. **DepEd 20% absence rule** uses wrong threshold (10 absences instead of 20%)
3. **attendance_patterns** table is not auto-populated (no batch job)
4. **Attendance rate formulas** mix day-based and period-based inconsistently
5. **Subject attendance** status is stored in `attendance_logs.status` without clear subject-specific status column
6. **Holiday/suspension handling** has grade-level filtering bugs
7. **Half-day computation** is approximate, not schedule-aware

## 2. Homeroom Attendance Flow (Gate → Daily Summary)

### Expected Flow (from spec)

1. **Gate Scan (ENTRY)** → Insert/update `attendance_logs` row with `time_in`, status (On Time/Late based on grade_schedules)
2. **Gate Scan (EXIT)** → Update same row's `time_out`, potentially flag early/late exit
3. **End of Day Batch** → For each student with scans, compute:
   - Look up `grade_schedules.start_time` and `late_threshold` for student's grade
   - Determine `morning_status` based on earliest `time_in` (no time_in → Absent)
   - Determine `afternoon_status` based on presence of afternoon scan OR subject attendance
   - Upsert into `attendance_daily_summary`
4. **Teacher Override** → Teacher manually edits `attendance_daily_summary` via homeroom dashboard (overrides auto)

### Current Implementation

**Gate Scanning (guard-core.js, teacher-gatekeeper-mode.js)**
- ✅ Scans QR → inserts `attendance_logs` with `time_in` (ENTRY) and updates `time_out` (EXIT)
- ✅ Status calculated: 'On Time' / 'Late' / 'Early Exit' / 'Late Exit' / 'Normal Exit'
- ✅ Duplicate scan detection (5-second cooldown)
- ✅ Real-time notification to parent & homeroom teacher
- ✅ Weekend/holiday/suspension pre-checks (block entry if suspended)
- ⚠️ **ISSUE:** Exit scans < 9 AM are blocked (`direction === 'EXIT' && currentHour < 9`) — this may incorrectly block legitimate early dismissals

**Daily Summary Population (attendance-utils.js, teacher-homeroom.js)**
- ✅ `recomputeHalfDayStatus(studentId, date)` exists and is called after subject attendance saves
- ✅ It queries subject loads, checks hour <12 / >=12 to set `morning_absent` / `afternoon_absent` booleans
- ✅ Upserts to `attendance_daily_summary` with `morning_status` / `afternoon_status` derived from flags
- ❌ **GAP:** No standalone end-of-day batch job that processes ALL students, even those with no scans (should mark them Absent)
- ❌ **GAP:** `attendance_daily_summary` entries are only created when:
  1. Subject attendance is saved (triggers recomputeHalfDayBatch) OR
  2. Homeroom teacher manually saves attendance (teacher-homeroom.js: `saveAllPending()` upserts to `attendance_logs`, but does NOT update `attendance_daily_summary`)
- **Result:** Students with only gate scans get summary entries; students with neither gate nor subject marks are **missing from summary table entirely**

**Teacher Override (teacher-homeroom.js)**
- ✅ Teacher can cycle status: blank → On Time → Late → Absent → Excused → Excused Absent
- ✅ Manual changes are saved directly to `attendance_logs` (not `attendance_daily_summary`)
- ❌ **GAP:** Manual changes do NOT trigger `recomputeHalfDayBatch`, so `attendance_daily_summary` becomes stale
- ❌ **GAP:** The override logic edits `attendance_logs` (raw logs) rather than `attendance_daily_summary`. The spec says teacher edits `attendance_daily_summary.morning_status`.

**Verdict:** ⚠️ Partial — Daily summary is recomputed only for students with subject attendance saves, not comprehensively. Override mechanism targets wrong table.

- Status: `Present` / `Late` / `Absent` / `Excused`
- Subject attendance counts toward **period-based attendance rate**
- Must respect `schedule_days` (only on scheduled days)

### Current

**Teacher Manual Entry (teacher-subject-attendance.js)**
- ✅ Teacher selects subject → loads students → marks attendance via color-square UI
- ✅ Saves to `attendance_logs` with `subject_load_id` and `status`
- ✅ Inserts new rows or updates existing for that date+student+subject
- ✅ Calls `recomputeHalfDayStatus` for each saved student (via `recomputeHalfDayBatch`)

**Status Values** (teacher-subject-attendance.js:85-90, teacher-homeroom.js:85-91)

**Verdict:** ✅ Subject attendance functional but status naming differs from spec. No gate-scan-as-homeroom-subject mechanism.

## 4. DepEd 20% Absence Rule & attendance_patterns

### Expected

- Count **unexcused absent days** (not periods) for each student
- If `unexcused_absent_days / total_school_days >= 20%` → flag as **critical**
- Insert into `attendance_patterns` with `pattern_type='critical_absences'`, `severity='high'`
- Create notifications for admin & homeroom teacher
- Admin can mark student as `Dropped`

### Current

**Detection (teacher-homeroom-table.js: checkAndNotifyHighAbsences)**
- ❌ Uses **fixed threshold of 10 absences YTD**, not 20% of school days
- ❌ Only counts `status === 'Absent'` OR `morning_absent && afternoon_absent` as full-day absence
- ❌ No use of `attendance_patterns` table — only sends notifications
- ❌ Not a batch job — runs only when teacher opens homeroom table page (on-demand)
- ❌ No admin-side critical list (admin-data-analytics.js: `fetchCriticalAbsences` uses threshold of `adjustedAbsence > 10`)

**Admin Analytics (admin-data-analytics.js: fetchCriticalAbsences)**
- ❌ Threshold: `s.absent + (s.halfday * 0.5) > 10` → "critical"
- ❌ Also does not use `attendance_patterns` table

**Verdict:** ❌ Major gap — **20% rule not implemented**. System uses hardcoded 10-absence threshold and does not store patterns.

## 5. Attendance Rate Calculations

### Expected Formulas

**Per-Student Attendance Rate:**
```
total_expected_periods = (school_days) × (1 homeroom AM + [1 PM if applicable] + subjects_per_day)
effective_present_periods = count of logs where status IN ('Present','Late','Excused')
attendance_rate = (effective_present_periods / total_expected_periods) × 100
```
- **Excused absences count as present** for rate purposes
- **Late counts as present** (just tardy)
- Half-day counts as 0.5 present

### Current Implementation

**Parent Dashboard (parent-childs-attendance.js: calculateStatsAndChart → calculateAttendanceStats)**
- ✅ Calls `calculateAttendanceStats(studentId, year, month)` (defined elsewhere, likely in parent-utils.js)
- Shows present/late/absent/halfday counts and percentage

**Admin Analytics (admin-data-analytics.js)**
- `fetchStatusDistribution`: counts logs as Present/Late/Absent/Excused/HalfDay
  - HalfDay counts as 0.5 Present
  - Excused adds to Excused count but also counts as Present? Actually line 458: `if (isExcused) counts.Excused++; else if (isHalfDay) { counts.HalfDay++; counts.Present += 0.5; }`
  - So Excused does **not** increment Present line 458 — but excused should be present-equivalent for rate calc
- `fetchAverageAttendanceRate`: computes `(present / logs.length) × 100` where present includes OnTime+Late+Excused+0.5*halfday
  - ✅ That's correct
- `fetchClassPerformance`: computes per-class rate using same logic

**Teacher Analytics (teacher-data-analytics.js: loadPeriodStats)**
- Similar logic with half-day adjustments

**Verdict:** ⚠️ Inconsistent — Some functions count Excused as Present for rate; others separate Excused category. Should consistently treat Excused as Present-equivalent for **rate** calculation while keeping separate count for reporting.

## 6. Tap-in / Tap-out Logic (Gatekeeper)

### Expected Decision Table

| Scenario | Condition | Action |
|----------|-----------|--------|
| Tap in (entrance) | No `time_in` today | Record `time_in`, status = on-time/late based on grade schedule |
| Duplicate tap in | Second `time_in` within 5 min | Ignore |
| Tap out (exit) | Current time < `end_time` | Mark as `Early Exit` |
| Tap out | `end_time` < time < `end_time+30min` | `Normal Exit` |
| Tap out | time > `end_time+30min` | `Late Exit` |
| No tap out by EOD | `time_out` is null | Set `afternoon_status = Absent`, notify parent |

### Current Implementation (guard-core.js: handleScan)

**Entry**
- ✅ Checks for existing log today: if none → ENTITY; if has time_in but no time_out → EXIT; if both present → new ENTRY (re-entry)
- ⚠️ **Re-entry logic**: creates a new record instead of updating existing; status becomes 'Re-entry'/'Late Re-entry' — not in spec
- ✅ Duplicate cooldown: 5 seconds (ANTI_DUPLICATE_THRESHOLD = 5000)

**Exit**
- ✅ If no `time_out` on existing log → update it
- ✅ If complete log exists → creates new exit record (line 700-709) — questionable

**Status Calculation (calculateStatus)**
- ✅ ENTRY: `isLate(scanTime, lateThreshold)` → 'Late' else 'On Time'
- ✅ EXIT: `isEarlyExit(scanTime, dismissalTime)` → 'Early Exit'; `isLateExit(...)` → 'Late Exit'; else 'Normal Exit'
- ✅ Times fetched from `grade_schedules` via `getLateThreshold()` / `getDismissalTime()`

**Early Exit Block**
- ❌ **BUG:** Blocks exit scans before 9 AM (`if (direction === 'EXIT' && currentHour < 9)`). This prevents legitimate early dismissals (e.g., clinic passes) before 9 AM.
- The spec says early exit is determined by comparing scan time with dismissal time, not a fixed hour.

**Verdict:** ⚠️ Mostly functional but has **blocking bug** for early exits before 9 AM and **re-entry creates duplicate rows** instead of logging new entry session.

## 10. Attendance Patterns & Critical Alerts (DepEd 20% Rule)

### Expected (Section 5 of spec)

1. Compute **total school days** in term (from `school_year_start` to `school_year_end`, minus weekends/holidays/suspensions)
2. Count student's **unexcused absent days** (days where `morning_status='Absent'` AND no approved excuse, or full-day absence)
3. If `(unexcused_absent_days / total_school_days) >= 20%`:
   - Insert into `attendance_patterns` (`pattern_type='critical_absences'`, `severity='high'`)
   - Create notification for **admin** and **homeroom teacher**
   - Admin UI can mark student as `Dropped`

### Current Implementation

**No `attendance_patterns` inserts anywhere** — confirmed by grep search. The table exists but is unused.

**Threshold used:** Fixed **10 absences** (teacher-homeroom-table.js: checkAndNotifyHighAbsences, admin-data-analytics.js: fetchCriticalAbsences threshold `> 10`). This is **not 20%** of school days.

**Batch job:** None. Detection runs only when teacher opens homeroom table page (on-demand).

**Verdict:** ❌ **Not implemented correctly** — wrong threshold, no persistence, no scheduled job.

```javascript
                    last_modified_by: currentTeacherId,
                    last_modified_at: new Date().toISOString()
                }, { onConflict: 'student_id, date' });

            if (error) throw error;

            // 3. Optional: also reflect in attendance_logs for audit trail
            // Check if a log exists for today (homeroom scan)
            const { data: existingLog } = await supabase
                .from('attendance_logs')
                .select('id')
                .eq('student_id', studentId)
                .eq('log_date', today)
                .is('subject_load_id', null)
                .maybeSingle();

            if (existingLog) {
                await supabase
                    .from('attendance_logs')
                    .update({ status: newStatus || 'Present' })
                    .eq('id', existingLog.id);
            } else {
                // Create a manual log entry for tracking
                await supabase.from('attendance_logs').insert({
                    student_id: parseInt(studentId),
                    log_date: today,
                    status: newStatus || 'Present',
                    remarks: `Manual override by teacher ${currentTeacherId}`,
                    subject_load_id: null
                });
            }

            // 4. Recompute half-day flags for this student/date
            if (typeof recomputeHalfDayStatus === 'function') {
                await recomputeHalfDayStatus(parseInt(studentId), today);
                anyRecomputed = true;
            }

            // 5. Notify parent if changed to Absent
            await notifyParentOnManualChange(studentId, today, newStatus);
        }

        // Refresh the current view
        await fetchAttendanceForDate();
        pendingUpdates = {};
        renderChecklist();
        showNotification('Homeroom attendance successfully saved!', 'success');

    } catch (err) {
        console.error('Error saving attendance:', err);
        showNotification('Error saving attendance: ' + err.message, 'error');
    }
```

```javascript
/**
 * Daily Summary Batch Job
 * 
 * This ensures attendance_daily_summary has one row per student per school day.
 * It fills gaps for students who never had a scan nor subject attendance that day.
 * 
 * Deployment options:
 * 1. Supabase Edge Function with pg_cron (recommended)

import { supabase } from './assets/supabase-client.js';

/**
 * Check if a date is a valid school day (weekend/holiday/suspension aware)
 */
async function isSchoolDay(dateStr) {
    const dayOfWeek = new Date(dateStr).getDay();
    if (dayOfWeek === 0) return false; // Sunday

    // Check Saturday setting
    if (dayOfWeek === 6) {
        const { data } = await supabase
            .from('settings')
            .select('setting_value')
            .eq('setting_key', 'weekend_saturday_enabled')
            .single();
        if (!data || data.setting_value !== 'true') return false;
    }

    // Check holidays
    const { data: holiday } = await supabase
        .from('holidays')
        .select('is_suspended, time_coverage')
        .eq('holiday_date', dateStr)
        .maybeSingle();
    if (holiday && holiday.is_suspended && holiday.time_coverage === 'Full Day') {
        return false;
    }

    // Check suspensions
    const { data: suspension } = await supabase
        .from('suspensions')
        .select('id')
        .eq('is_active', true)
        .lte('start_date', dateStr)
        .gte('end_date', dateStr)
        .maybeSingle();
    if (suspension) return false;

    return true;
}

/**
 * Compute morning/afternoon status from gate scans and subject attendance
 */
async function computeStudentDailyStatus(studentId, date) {
    // 1. Check gate scans (no subject_load_id)
    const { data: gateLogs } = await supabase
        .from('attendance_logs')
        .select('time_in, status')
        .eq('student_id', studentId)
        .eq('log_date', date)
        .is('subject_load_id', null)
        .order('time_in', { ascending: true });

    let morningStatus = 'Absent';
    let afternoonStatus = 'Absent';

    if (gateLogs && gateLogs.length > 0) {
        // Use earliest gate scan for morning determination
        const earliest = gateLogs[0];
        const hour = new Date(earliest.time_in).getHours();
        if (hour < 12) {
            morningStatus = earliest.status === 'Late' ? 'Late' : 'Present';
        } else {
            afternoonStatus = earliest.status === 'Late' ? 'Late' : 'Present';
        }

        // Multiple gate entries (re-entry): use first for AM, last for PM if after 12
        const lastGate = gateLogs[gateLogs.length - 1];
        if (lastGate.time_in && new Date(lastGate.time_in).getHours() >= 12) {
            afternoonStatus = lastGate.status === 'Late' ? 'Late' : 'Present';
        }
    }

    // 2. Check subject attendance to override (a single subject attended = present for that half)
    const { data: subjectLogs } = await supabase
        .from('attendance_logs')
        .select('subject_load_id, time_in')
        .eq('student_id', studentId)
        .eq('log_date', date)
        .not('subject_load_id', 'is', null);

    if (subjectLogs && subjectLogs.length > 0) {
        for (const log of subjectLogs) {
            const hour = log.time_in ? new Date(log.time_in).getHours() : 12;
            if (hour < 12 && morningStatus === 'Absent') {
                morningStatus = 'Present';  // Attended a morning subject
            }
            if (hour >= 12 && afternoonStatus === 'Absent') {
                afternoonStatus = 'Present';  // Attended an afternoon subject
            }
        }
    }

    return { morningStatus, afternoonStatus };
}

/**
 * Sync daily summary for a single date
 */
export async function syncDailySummaryForDate(dateStr) {
    console.log(`[DailySummaryBatch] Starting sync for ${dateStr}`);

    // 0. Check if school day
    if (!(await isSchoolDay(dateStr))) {
        console.log(`[DailySummaryBatch] ${dateStr} is not a school day – skipping`);
        return;
    }

    // 1. Get all enrolled students
    const { data: students, error } = await supabase
        .from('students')
        .select('id, full_name, class_id')
        .eq('status', 'Enrolled');

    if (error || !students) {
        console.error('Failed to fetch students:', error);
        return;
    }

    console.log(`[DailySummaryBatch] Processing ${students.length} students`);

    let created = 0, updated = 0;

    // Process in batches to avoid overwhelming DB
    const batchSize = 100;
    for (let i = 0; i < students.length; i += batchSize) {
        const batch = students.slice(i, i + batchSize);

        for (const student of batch) {
            // Compute status
            const { morningStatus, afternoonStatus } = await computeStudentDailyStatus(student.id, dateStr);

            // Upsert into attendance_daily_summary
            const { data: existing } = await supabase
                .from('attendance_daily_summary')
                .select('id')
                .eq('student_id', student.id)
                .eq('date', dateStr)
                .maybeSingle();

            if (existing) {
                // Update if different
                const { error: updErr } = await supabase
                    .from('attendance_daily_summary')
                    .update({
                        morning_status: morningStatus,
                        afternoon_status: afternoonStatus,
                        last_modified_by: 'system-batch',
                        last_modified_at: new Date().toISOString()
                    })
                    .eq('id', existing.id);
                if (!updErr) updated++;
            } else {
                // Insert new
                const { error: insErr } = await supabase
                    .from('attendance_daily_summary')
                    .insert({
                        student_id: student.id,
                        date: dateStr,
                        morning_status: morningStatus,
                        afternoon_status: afternoonStatus,
                        last_modified_by: 'system-batch',
                        last_modified_at: new Date().toISOString()
                    });
                if (!insErr) created++;
            }
        }

        // Small delay between batches
        await new Promise(resolve => setTimeout(resolve, 100));
    }

    console.log(`[DailySummaryBatch] Complete: ${created} created, ${updated} updated`);
}
```

### Task 4: Implement DepEd 20% Absence Rule

**File:** `attendance-rules.js` (create at project root or in `core/`)

```javascript
/**
 * DepEd 20% Absence Rule Implementation
 * 
 * Business Rule: A student must be dropped if total unexcused absences
 * exceed 20% of total school days in the current school year.
 * 
 * This module:
 * 1. Calculates total school days (excludes weekends, holidays, suspensions)
 * 2. Counts unexcused absent days per student
 * 3. Inserts critical pattern records into attendance_patterns
 * 4. Sends notifications to admin and homeroom teacher
 */

import { supabase } from './assets/supabase-client.js';

/**
 * Get total school days between two dates, optionally filtered by grade level.
 * Counts only days when classes are in session (Mon-Fri excluding holidays/suspensions).
 */
async function getTotalSchoolDays(startDate, endDate, gradeLevel = null) {
    const start = new Date(startDate);
    const end = new Date(endDate);
    let count = 0;

    // Pre-fetch holidays for efficiency
    const { data: holidays } = await supabase
        .from('holidays')
        .select('holiday_date, target_grades, time_coverage')
        .gte('holiday_date', startDate)
        .lte('holiday_date', endDate)
        .eq('is_suspended', true);

    // Pre-fetch suspensions
    const { data: suspensions } = await supabase
        .from('suspensions')
        .select('start_date, end_date, affected_grades, saturday_enabled')
        .eq('is_active', true)
        .lte('start_date', endDate)
        .gte('end_date', startDate);

    const current = new Date(start);
    while (current <= end) {
        const dayOfWeek = current.getDay();
        const dateStr = current.toISOString().split('T')[0];

        // Skip Sunday
        if (dayOfWeek === 0) {
            current.setDate(current.getDate() + 1);
            continue;
        }

        // Saturday handling
        if (dayOfWeek === 6) {
            const saturdayClassEnabled = (await supabase
                .from('settings')
                .select('setting_value')
                .eq('setting_key', 'weekend_saturday_class_enabled')
                .single()).data?.setting_value === 'true';

            const hasSaturdaySuspension = suspensions?.some(s =>
                new Date(s.start_date) <= current &&
                new Date(s.end_date) >= current &&
                s.saturday_enabled !== true  // If suspension has saturday_enabled=false, it overrides
            );

            if (!saturdayClassEnabled || hasSaturdaySuspension) {
                current.setDate(current.getDate() + 1);
                continue;
            }
        }

        // Holiday full-day check
        const holiday = holidays?.find(h => h.holiday_date === dateStr);
        if (holiday) {
            if (holiday.target_grades && holiday.target_grades !== 'All' && gradeLevel) {
                const affected = holiday.target_grades.split(',').map(g => g.trim());
                if (!affected.includes(gradeLevel)) {
                    // This grade not affected → still a school day
                } else if (holiday.time_coverage === 'Full Day') {
                    current.setDate(current.getDate() + 1);
                    continue;
                }
                // Morning/Afternoon partial holidays still count as school days
            } else if (holiday.time_coverage === 'Full Day') {
                current.setDate(current.getDate() + 1);
                continue;
            }
        }

        // Suspension full-day check
        const daySuspension = suspensions?.find(s =>
            new Date(s.start_date) <= current &&
                    new Date(s.end_date) >= current
                );
        if (daySuspension) {
            const affectsThisGrade = !daySuspension.affected_grades?.length ||
                daySuspension.affected_grades.includes(gradeLevel);
            if (affectsThisGrade && daySuspension.suspension_type !== 'saturday_class') {
                current.setDate(current.getDate() + 1);
                continue;
            }
        }

        count++;
        current.setDate(current.getDate() + 1);
    }

    return count;
}

/**
 * Count unexcused absent days for a student in a date range.
 * Uses attendance_daily_summary as the source of truth.
 */
async function countUnexcusedAbsentDays(studentId, startDate, endDate) {
    const { data: summaries } = await supabase
        .from('attendance_daily_summary')
        .select('date, morning_status, afternoon_status')
        .eq('student_id', studentId)
        .gte('date', startDate)
        .lte('date', endDate);

    if (!summaries || summaries.length === 0) return 0;

    // Fetch approved excuse letters for this student
    const { data: excuses } = await supabase
        .from('excuse_letters')
        .select('date_absent, absence_type, period')
        .eq('student_id', studentId)
        .eq('status', 'Approved')
        .gte('date_absent', startDate)
        .lte('date_absent', endDate);

    // Build a map: date → { absence_type, period }
    const excuseMap = new Map();
    excuses?.forEach(e => {
        excuseMap.set(e.date_absent, {
            type: e.absence_type || 'whole_day',
            period: e.period || 'whole_day'
        });
    });

    let unexcusedCount = 0;

    for (const summary of summaries) {
        const excuse = excuseMap.get(summary.date);

        // If fully excused, skip
        if (excuse) {
            if (excuse.type === 'whole_day') continue;
            if (excuse.type === 'half_day_morning' && summary.morning_status === 'Absent') continue;
            if (excuse.type === 'half_day_afternoon' && summary.afternoon_status === 'Absent') continue;
        }

        const morningAbsent = summary.morning_status === 'Absent';
        const afternoonAbsent = summary.afternoon_status === 'Absent';

        // Both absent = full day absence (count = 1)
        if (morningAbsent && afternoonAbsent) {
            unexcusedCount += 1;
        }
        // Only one session absent = half day (count = 0.5)
        else if (morningAbsent || afternoonAbsent) {
            unexcusedCount += 0.5;
        }
        // Both present = no absence (count = 0)
    }

    return unexcusedCount;
}

/**
 * Check a single student and flag if critical
 */
export async function checkCriticalAbsencesForStudent(studentId, gradeLevel, schoolYearStart, schoolYearEnd) {
    const totalDays = await getTotalSchoolDays(schoolYearStart, schoolYearEnd, gradeLevel);
    if (totalDays === 0) {
        console.warn(`No school days found for ${gradeLevel} between ${schoolYearStart}–${schoolYearEnd}`);
        return false;
    }

    const unexcusedDays = await countUnexcusedAbsentDays(studentId, schoolYearStart, schoolYearEnd);
    const absenceRate = unexcusedDays / totalDays;

    console.log(`[DepEdRule] Student ${studentId}: ${unexcusedDays} unexcused days / ${totalDays} total = ${(absenceRate * 100).toFixed(2)}%`);

    if (absenceRate >= 0.20) {
        // Check if pattern already exists (unresolved)
        const { data: existing } = await supabase
            .from('attendance_patterns')
            .select('id')
            .eq('student_id', studentId)
            .eq('pattern_type', 'critical_absences')
            .eq('is_resolved', false)
            .maybeSingle();

        if (!existing) {
            // Insert new pattern record
            const { error: patErr } = await supabase.from('attendance_patterns').insert({
                student_id: studentId,
                pattern_type: 'critical_absences',
                description: `Absence rate ${(absenceRate * 100).toFixed(1)}% exceeds DepEd 20% threshold (${unexcusedDays}/${totalDays} days).`,
                severity: 'high',
                is_resolved: false,
                created_at: new Date().toISOString()
            });
            if (patErr) console.error('Failed to insert attendance_patterns:', patErr);

            // Notify admin(s)
            const { data: admins } = await supabase
                .from('admins')
                .select('id')
                .eq('is_active', true);
            for (const admin of admins || []) {
                await supabase.from('notifications').insert({
                    recipient_id: admin.id,
                    recipient_role: 'admin',
                    title: '⚠️ Critical Absences Alert',
                    message: `Student has exceeded the DepEd 20% absence threshold (${(absenceRate * 100).toFixed(1)}%). Review required.`,
                    type: 'absence_alert',
                    is_urgent: true,
                    created_at: new Date().toISOString()
                });
            }

            // Notify homeroom teacher
            const { data: studentRec } = await supabase
                .from('students')
                .select('class_id, full_name')
                .eq('id', studentId)
                .single();

            if (studentRec?.class_id) {
                const { data: classInfo } = await supabase
                    .from('classes')
                    .select('adviser_id')
                    .eq('id', studentRec.class_id)
                    .single();

                if (classInfo?.adviser_id) {
                    await supabase.from('notifications').insert({
                        recipient_id: classInfo.adviser_id,
                        recipient_role: 'teacher',
                        title: '⚠️ Critical Absences Alert',
                        message: `Student ${studentRec.full_name} in your class has exceeded the DepEd 20% absence threshold (${(absenceRate * 100).toFixed(1)}%). Please intervene.`,
                        type: 'absence_alert',
                        is_urgent: true,
                        created_at: new Date().toISOString()
                    });
                }
            }

            console.log(`[DepEdRule] Flagged student ${studentId} as critical (${(absenceRate * 100).toFixed(1)}%)`);
            return true;
        }
    }

    return false;
}
```
