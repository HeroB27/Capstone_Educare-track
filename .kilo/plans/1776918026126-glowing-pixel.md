# Implementation Plan: Attendance & Data Analytics Logic

## Overview
This plan implements the attendance and data analytics logic as specified in the attendance_and_data_analytics.md plan. The goal is to migrate all analytics from using `attendance_logs` to using `attendance_daily_summary` as the source of truth, while implementing proper Kinder exclusion, DepEd 20% rule, and other attendance logic corrections.

## Current State Analysis
Based on code review:
- `core/attendance-helpers.js` exists but does NOT contain the required helper functions (getTotalSchoolDays, countUnexcusedAbsentDays, getAttendanceRate)
- `core/analytics-shared-core.js` contains the required analytics helper functions:
  - getTotalSchoolDays(startDate, endDate, gradeLevel) - excludes Sundays, full-day holidays, suspensions
  - fetchSummaryStats(studentIds, startDate, endDate, gradeLevel) - fetches from attendance_daily_summary
  - fetchCriticalAbsences(studentIds, startDate, endDate, gradeLevel) - implements DepEd 20% rule
  - fetchAverageAttendanceRate(studentIds, startDate, endDate, gradeLevel) - calculates average attendance rate
- `admin/admin-data-analytics.js` and `teacher/teacher-data-analytics.js` exist but still use `attendance_logs` for some functions
- Some functions have been partially migrated but need completion
- Nightly batch jobs (`attendance-daily-summary-batch.js`, `attendance-rules.js`) exist

## Success Criteria

After implementation:
1. All analytics data (trend charts, status distribution, critical absences, average attendance) reads from `attendance_daily_summary`
2. Kinder students' afternoon sessions are excluded from all analytics calculations
3. DepEd 20% rule uses proper school day calculation (excluding Sundays, holidays, suspensions)
4. Excused absences count as present in all calculations
5. Nightly batch jobs maintain synchronization between logs and summary tables
6. Analytics match manual calculations based on the documented attendance logic

## Implementation Order

1. Verify shared helper functions in `core/analytics-shared-core.js` are correct and being used
2. Update admin analytics functions to use `attendance_daily_summary` via shared core functions
3. Update teacher analytics functions to use `attendance_daily_summary` via shared core functions
4. Implement feature flag for gradual rollout

6. Test with sample data to validate correctness
