# Attendance & Notifications Enhancement Plan

## Goal
Focus on the attendance system and notifications for Admin, Teacher, and Parent roles. Excluding RLS and Auth features.

---

## User Decisions (Confirmed)

- **Late Notifications:** All late arrivals trigger notification
- **Early Dismissal:** Immediate notification (no approval required)
- **Architecture:** Extend `notification-engine.js` for all events (centralized)

---

## Current State Analysis

### ✅ Already Implemented

**Gate Notifications:**
- `guard/guard-core.js` creates notifications on scan (lines 585-597)
- Types: `arrival` for entry, `departure` for exit
- Parent receives "Arrival Alert" / "Departure Alert"

**Teacher Notifications:**
- Late/Early Exit/Late Exit alerts sent to homeroom teacher (line 610)
- Status types: `attendance_alert`

**Attendance System:**
- Homeroom attendance with colored status squares
- Subject attendance per class
- Half-day (AM/PM) partial absence detection
- Gate scan integration with real-time updates

**Excuse Letter:**
- Status display in parent portal (line 142 of parent-excuse-letter-template.js)
- Real-time updates via subscription

### ❌ Actually Missing

1. **Late/Early Notifications to Parents** - Teachers get notified (line 610), but **parents do NOT** when student arrives late or leaves early
2. **Clinic Visit Parent Notification** - Database has `parent_notified` column but not triggered
3. **Notification Type Mismatch** - Gate uses `arrival`/`departure`, parent filter expects `gate_entry`/`gate_exit`

---

## Implementation Plan

### Phase 1: Fix Notification Types (Quick Fix)
**Files:** `guard/guard-core.js`

Change notification types from `arrival`/`departure` to `gate_entry`/`gate_exit` to match parent filter expectations.

```javascript
// Line 595 in guard-core.js
type: direction === 'ENTRY' ? 'gate_entry' : 'gate_exit'
```

### Phase 2: Late/Early Notifications to Parents
**Files:** `guard/guard-core.js` - extend `createNotification()`

When status is Late/Early Exit/Late Exit, also notify parent:
- Add parent notification for late arrivals
- Add parent notification for early exits

### Phase 3: Clinic Visit Parent Notification
**Files:** `clinic/clinic-core.js`

On clinic visit creation:
- Get parent_id from student
- Insert notification with type 'clinic'
- Include reason, nurse notes, action taken
- Set `parent_notified = true` in clinic_visits
