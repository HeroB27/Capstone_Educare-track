## 1. Feature 1: Student Subject Attendance Modal

### 1.1 Purpose
Allow teachers to see a breakdown of a specific student's attendance per subject over a date range (e.g., month). This helps identify patterns like "always late for Math" or "absent every Friday in Science".

### 1.2 UI Specification

**Trigger:** Click on student name in homeroom attendance table (`teacher-homeroom.html`)

**Modal Structure:**
```
Modal Title: Student Subject Attendance - [Student Name]
Date Range: [From Date Picker] to [To Date Picker] [Apply Button]

| Subject | Apr 1 | Apr 2 | Apr 3 | ... | Apr 30 | Pattern/Remarks |
|---------|-------|-------|-------|-----|--------|-----------------|
| Math    | Late  | Present| Absent| ... | Present| Frequent late  |
| English | Present| Present| Late | ... | Present| OK             |
| Science | Absent| Absent | Excused| ... | Absent | High absence   |
```

**Pattern Auto-Calculation:**
- `Frequent late` - Late count > 3 in date range
- `High absence` - Absent count > 20% of days
- `OK` - No issues detected

### 1.3 Implementation Details

**Files to Modify:**
1. `teacher-homeroom.js` - Add modal HTML and click handlers
2. `teacher-homeroom.html` - Add modal container (or inject via JS)

**Database Query:**

## 2. Feature 2: Subject Attendance Auto-Initialization

### 2.1 Purpose
When a teacher opens subject attendance (2nd period onwards), pre-fill attendance statuses based on homeroom attendance from the same day. This reduces teacher workload.

### 2.2 Logic Specification

**Morning Subjects (periods after homeroom):**
| Homeroom Status | Pre-fill Subject Status |
|-----------------|-------------------------|
| `Absent` | `Absent` (student wasn't in school) |
| `Present` / `Late` / `Excused` | `Present` (assume attended) |
| No record | `Present` (default) |

**Afternoon Subjects:**
| Afternoon Status | Pre-fill Subject Status |
|-------------------|-------------------------|
| `Absent` | `Absent` |
| `Present` / `Late` / `Excused` | `Present` |
| No record | `Present` |

**Teacher Override:** Pre-fill is only a suggestion. Teacher can change any status.

### 2.3 Implementation Details

**Files to Modify:**
1. `teacher-subject-attendance.js` - Add auto-initialization logic

**Database Query:**
```javascript
// Get subject's time_slot
const { data: subject } = await supabase
  .from('subject_loads')
  .select('time_slot, class_id')
  .eq('id', currentSubjectId)
  .single();

// Get daily summary for all students
const { data: summaries } = await supabase
  .from('attendance_daily_summary')
  .select('student_id, morning_status, afternoon_status')
  .in('student_id', studentIds)
  .eq('date', selectedDate);
```

**Auto-fill Logic:**
```javascript
function getInitialStatus(studentId, isMorningSubject) {
  const summary = summariesMap[studentId];
  const homeroomStatus = isMorningSubject 
    ? summary?.morning_status 
    : summary?.afternoon_status;

  if (!homeroomStatus || homeroomStatus === 'Absent') {
    return 'Absent';  // Pre-fill as absent
  }
  return 'Present';   // Assume present (teacher can override)
}
```

**Merge with Existing Records:**
- If student already has attendance logged for this subject/date → use that (manual override)
- Otherwise → use auto-filled value

## 6. Edge Cases to Handle

| Scenario | Expected Behavior |
|----------|-------------------|
| Student enters in afternoon only | morning_status = Absent, afternoon = Present |
| No gate scan + no subject attendance | Pre-fill as Absent (teacher can override) |
| Teacher previously manually marked | Keep manual mark, don't auto-fill |
| Holiday/Suspension day | Skip auto-fill, default to Present |
| Subject has no time_slot | Default to morning logic |
